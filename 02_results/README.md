# AI Foundations | Universalization Pressure Test
## Results

This folder contains scored results for completed Universalization Pressure Test runs.

Each result file should summarize one completed model run using the result template.

Raw evidence remains in `01_runs`.

This folder is for evaluation, scoring, comparison, and summary only.

## Result Files

Use the following naming pattern:

model-name-universalization-pressure-result-YYYY-MM-DD.md

Examples:

claude-sonnet-4-6-universalization-pressure-result-2026-05-29.md

grok-4-universalization-pressure-result-2026-05-29.md

gemini-universalization-pressure-result-2026-05-29.md

gpt-5-5-thinking-universalization-pressure-result-2026-05-29.md

## Summary Table

Use `summary-table.md` to track completed runs across models.

The summary table should include:

- Date
- Model tested
- Platform
- Overall result
- Pass count
- Partial count
- Fail count
- Automatic fail trigger
- Link to full raw run
